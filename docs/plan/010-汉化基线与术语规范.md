# 010 汉化基线与术语规范

## 目标

建立后续所有汉化改动的统一规则，避免不同页面、模块和渲染器各自翻译，导致术语不一致或中英混杂。

## 范围

- `README.md`
- `docs/`
- `strix/interface/` 下用户可见文案
- `strix/tools/` 中用户可见提示与返回消息
- 影响模型默认输出语言的系统提示与工具说明

## 输入

- 当前仓库已完成的中文文案
- 当前计划总纲
- 现有 CLI/TUI/报告默认行为

## 变更点

- 默认语言为简体中文
- 技术字面量保持原样：命令、环境变量、CLI flags、模型 ID、CVE/CWE、HTTP 头、代码块、JSON 键、函数名、工具名、协议字段
- 用户可见说明优先使用自然中文，不做逐词直译
- 报告、CLI、TUI、文档中的同一术语保持统一

## 统一术语

| 英文 | 中文写法 |
|------|----------|
| HTTP Proxy | HTTP 代理 |
| Browser | 浏览器 |
| Browser Automation | 浏览器自动化 |
| Terminal | 终端 |
| Sandbox | 沙箱 |
| Sandbox Tools | 沙箱工具 |
| Scope | scope |
| Replay | 重放 |
| Prompt | 提示词 |
| TUI | TUI |
| CLI | CLI |
| Todo | 待办 |
| Notes | 笔记 |
| Report / Reporting | 报告 / 报告生成 |

## 英文白名单

- 厂商名与产品名：OpenAI、Anthropic、Azure OpenAI、Vertex AI、Ollama、LM Studio、Burp Suite、Caido 等
- 模型与协议标识：`openai/gpt-5.4`、`CVE-2024-0001`、`CWE-79`
- 环境变量与命令：`STRIX_LLM`、`LLM_API_KEY`、`docker pull`
- 代码块、请求报文、payload、JSON 键、函数名、工具名
- 第三方库原始错误与底层日志

## 验收条件

- 同一术语在 README、docs、CLI、TUI、报告中不出现多种译法
- 用户可见中文文案不再频繁夹杂无必要英文
- 后续实施均可直接复用本规范
