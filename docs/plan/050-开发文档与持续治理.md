# 050 开发文档与持续治理

## 目标

把维护者日常会阅读的开发文档收口为中文，并把“新增用户可见文案默认中文”的规则沉淀进贡献流程。

## 范围

- 开发者说明文档
- 工具 README
- skills 入口说明
- 贡献规范中的汉化要求

## 输入

- 010 术语规范
- 现有开发文档与贡献文档

## 变更点

- 把开发者常读文档改为中文优先
- 在贡献规范中增加“新增用户可见文案默认中文”的要求
- 在计划索引中加入新增页面/新功能的汉化检查清单
- 不处理纯内部源码注释、测试名和第三方引用文档

## 验收条件

- 维护者常读文档默认中文
- 新增功能在进入主线前有明确的汉化检查要求
- 后续不再出现“文档中文、运行时全英文”的明显回退

## 剩余问题

- 部分仅供内部调试的注释与日志仍保留英文，不纳入本轮执行面
