---
title: "GitHub Actions"
description: "在每个 Pull Request 上运行 Strix 安全扫描"
---

将 Strix 集成到 GitHub 工作流中，在漏洞进入生产环境前完成拦截。

## 基础工作流

```yaml .github/workflows/security.yml
name: Security Scan

on:
  pull_request:

jobs:
  strix-scan:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0

      - name: Install Strix
        run: curl -sSL https://strix.ai/install | bash

      - name: Run Security Scan
        env:
          STRIX_LLM: ${{ secrets.STRIX_LLM }}
          LLM_API_KEY: ${{ secrets.LLM_API_KEY }}
        run: strix -n -t ./ --scan-mode quick
```

## 必需的 Secrets

请在仓库中配置以下 Secrets：

| Secret | 说明 |
|--------|------|
| `STRIX_LLM` | 模型名，例如 `openai/gpt-5.4` |
| `LLM_API_KEY` | LLM 提供商的 API Key |

## 退出码

如果发现漏洞，工作流会失败：

| 代码 | 结果 |
|------|------|
| 0 | 通过，未发现漏洞 |
| 2 | 失败，发现漏洞 |

## 适用于 CI 的扫描模式

| 模式 | 耗时 | 使用场景 |
|------|------|----------|
| `quick` | 数分钟 | 每个 PR |
| `standard` | 约 30 分钟 | 夜间构建 |
| `deep` | 1 到 4 小时 | 发版候选版本 |

<Tip>
建议在 PR 中使用 `quick` 模式，以保持反馈足够快；`deep` 扫描更适合夜间或定时任务。
</Tip>

<Note>
对于 `pull_request` 工作流，Strix 会在 CI / 无头模式下自动启用 changed-files diff-scope。如果 diff 解析失败，请确保已拉取完整历史（`fetch-depth: 0`），或者显式设置 `--diff-base`。
</Note>
