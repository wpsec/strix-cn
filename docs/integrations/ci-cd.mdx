---
title: "CI/CD 集成"
description: "在任意 CI/CD 流水线中运行 Strix"
---

Strix 支持以无头模式运行，适合自动化流水线。

## 无头模式

使用 `-n` 或 `--non-interactive`：

```bash
strix -n --target ./app --scan-mode quick
```

对于 Pull Request 风格的 CI 场景，Strix 会自动把快速扫描限制在变更文件内。你也可以显式强制这个行为，并指定基线引用：

```bash
strix -n --target ./app --scan-mode quick --scope-mode diff --diff-base origin/main
```

## 退出码

| 代码 | 含义 |
|------|------|
| 0 | 未发现漏洞 |
| 1 | 执行错误 |
| 2 | 发现漏洞 |

## GitLab CI

```yaml .gitlab-ci.yml
security-scan:
  image: docker:latest
  services:
    - docker:dind
  variables:
    STRIX_LLM: $STRIX_LLM
    LLM_API_KEY: $LLM_API_KEY
  script:
    - curl -sSL https://strix.ai/install | bash
    - strix -n -t ./ --scan-mode quick
```

## Jenkins

```groovy Jenkinsfile
pipeline {
    agent any
    environment {
        STRIX_LLM = credentials('strix-llm')
        LLM_API_KEY = credentials('llm-api-key')
    }
    stages {
        stage('Security Scan') {
            steps {
                sh 'curl -sSL https://strix.ai/install | bash'
                sh 'strix -n -t ./ --scan-mode quick'
            }
        }
    }
}
```

## CircleCI

```yaml .circleci/config.yml
version: 2.1
jobs:
  security-scan:
    docker:
      - image: cimg/base:current
    steps:
      - checkout
      - setup_remote_docker
      - run:
          name: Install Strix
          command: curl -sSL https://strix.ai/install | bash
      - run:
          name: Run Scan
          command: strix -n -t ./ --scan-mode quick
```

<Note>
所有 CI 平台都需要能访问 Docker。请确保你的 Runner 已提供 Docker 能力。
</Note>

<Tip>
如果 CI 中 diff-scope 解析失败，请拉取完整 Git 历史（例如 GitHub Actions 中设置 `fetch-depth: 0`），以便正确计算 merge-base 和分支差异。
</Tip>
