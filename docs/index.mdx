Strix 是一组自治 AI 安全代理，工作方式尽量贴近真实黑客。它会动态运行你的代码、发现漏洞，并通过 PoC 进行验证。`strix-cn` 在保留上游核心能力的同时，持续优先解决国内模型接入、Burp 工作流协同、环境兼容和中文化体验，方便在中文团队、内网环境和受限网络中落地。

<Info>
  `strix-cn` 是持续维护分支，当前文档会优先围绕四个方向演进：国内可用模型接入、Burp Suite 工作流兼容、跨平台与网络环境适配、中文文档与中文报告输出。
</Info>

## 使用场景

- **应用安全测试**：发现并验证应用中的关键安全漏洞
- **快速渗透测试**：将测试周期从几周压缩到几小时
- **漏洞赏金自动化**：自动化研究和 PoC 生成，加快提交速度
- **CI/CD 集成**：在漏洞进入生产前完成阻断
- **内网与受限环境落地**：配合本地模型、兼容网关与网络代理完成扫描

## 核心能力

- **完整黑客工具箱**：浏览器自动化、HTTP 代理、终端、Python 运行时
- **真实验证**：输出可执行 PoC，而不是误报告警
- **多代理协同**：专门化代理协作处理复杂目标
- **开发者优先 CLI**：既支持交互式 TUI，也支持自动化无头模式
- **模型兼容优先**：优先验证 OpenAI-compatible、LiteLLM 和本地模型接入
- **语言与环境适配**：默认中文报告，并持续修复跨平台兼容问题

## 安全工具

Strix 代理内置完整的安全测试工具链：

| 工具 | 作用 |
|------|------|
| HTTP Proxy | 基于 Caido 的请求 / 响应操作，并持续补齐 Burp 工作流兼容 |
| Browser Automation | 多标签浏览器，可测试 XSS、CSRF、认证流程 |
| Terminal | 用于命令执行的交互式 Shell |
| Python Runtime | 编写与验证自定义 PoC |
| Reconnaissance | 自动化 OSINT 与攻击面测绘 |
| Code Analysis | 同时支持静态与动态分析 |

## 漏洞覆盖范围

| 类别 | 示例 |
|------|------|
| 访问控制 | IDOR、提权、认证绕过 |
| 注入攻击 | SQL、NoSQL、命令注入 |
| 服务端漏洞 | SSRF、XXE、反序列化 |
| 客户端漏洞 | XSS、原型污染、DOM 漏洞 |
| 业务逻辑 | 竞态条件、流程操控 |
| 认证安全 | JWT 漏洞、会话管理问题 |
| 基础设施 | 配置错误、暴露服务 |

## 多代理架构

Strix 通过一张由专门化代理组成的图来完成更完整的安全测试：

- **分布式工作流**：不同代理分别负责不同攻击面和资产
- **可扩展测试**：并行执行以提升覆盖速度
- **动态协作**：代理之间共享发现、联动攻击路径

## 快速示例

```bash
# 安装
curl -sSL https://strix.ai/install | bash

# 配置兼容模型网关
export STRIX_LLM="openai/your-compatible-model"
export LLM_API_KEY="your-api-key"
export LLM_API_BASE="https://your-gateway.example/v1"

# 扫描
strix --target ./your-app
```

<Tip>
  当前分支默认输出中文报告，也支持直接使用中文 `--instruction` 编写测试要求。
</Tip>
