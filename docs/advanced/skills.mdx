Skills 是结构化的专业知识包，用来让 Strix 代理在特定漏洞类型、技术栈和测试方法上具备更深的上下文能力。

## 核心思路

LLM 的通识能力很强，但安全测试需要很多细节经验。模型通常“知道有 SQL 注入”，却不一定天然掌握经验型渗透测试人员常用的解析器特性、绕过技巧、验证手法和漏洞链路。

Skills 的作用，就是把这些高密度、可执行的专业知识直接注入代理上下文，让代理从“通用型助手”变成“当前任务上的专长型助手”。

## 工作方式

当 Strix 为某个具体任务创建代理时，会按上下文自动选择最多 5 个相关 skill：

```python
# 为 JWT 测试创建的代理会自动加载相关 skills
create_agent(
    task="Test authentication mechanisms",
    skills=["authentication_jwt", "business_logic"]
)
```

这些 skills 会被注入到代理的系统提示中，使它能直接使用：

- **进阶技巧**：超出标准检查项的高价值方法
- **可用 payload**：带变体的实战示例
- **验证方法**：如何确认漏洞并降低误报

## Skill 结构

每个 skill 都是带 YAML frontmatter 的 Markdown 文件：

```markdown
---
name: skill_name
description: skill 覆盖范围的简要说明
---

# Skill 标题

这里先写这个漏洞类型或技巧最关键的一条判断。

## 攻击面
说明这个 skill 主要覆盖哪些位置、哪些入口值得优先检查。

## 方法论
写清楚逐步测试流程和推荐顺序。

## 技术要点
整理发现、验证和利用该问题时的关键技巧。

## 绕过方法
记录常见防护的绕过思路与注意事项。

## 验证方式
说明如何确认问题真实存在，以及如何避免误报。
```

## 贡献 Skills

欢迎社区贡献新的 skills。请在合适的分类目录中创建带 YAML frontmatter 的 `.md` 文件。高质量 skill 通常包含：

1. **真实可用的方法**：不仅能说通，而且在实际环境里可复现
2. **实战 payload**：包含变体与边界条件
3. **验证步骤**：说明如何确认漏洞、避免误报
4. **上下文意识**：体现版本差异、部署环境差异和真实约束
