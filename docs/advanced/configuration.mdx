你可以通过环境变量或配置文件来配置 Strix。

## LLM 配置

<ParamField path="STRIX_LLM" type="string" required>
  以 LiteLLM 格式指定模型名，例如 `openai/gpt-5.4`、`anthropic/claude-sonnet-4-6` 或兼容网关下的 `openai/<your-model>`。
</ParamField>

<ParamField path="LLM_API_KEY" type="string">
  LLM 提供商的 API Key。本地模型或云厂商原生认证方式（如 Vertex AI、AWS Bedrock）可不填写。
</ParamField>

<ParamField path="LLM_API_BASE" type="string">
  自定义 API base URL。也接受 `OPENAI_API_BASE`、`LITELLM_BASE_URL` 或 `OLLAMA_API_BASE`。
</ParamField>

<ParamField path="LLM_TIMEOUT" default="300" type="integer">
  LLM 请求超时时间，单位为秒。
</ParamField>

<ParamField path="STRIX_LLM_MAX_RETRIES" default="5" type="integer">
  LLM API 短暂失败时的最大重试次数。
</ParamField>

<ParamField path="STRIX_REASONING_EFFORT" default="high" type="string">
  控制推理模型的思考强度。可选值为 `none`、`minimal`、`low`、`medium`、`high`、`xhigh`。在 `quick` 扫描模式下默认会降为 `medium`。
</ParamField>

<ParamField path="STRIX_MEMORY_COMPRESSOR_TIMEOUT" default="30" type="integer">
  上下文压缩操作的超时时间，单位为秒。
</ParamField>

## 可选功能

<ParamField path="PERPLEXITY_API_KEY" type="string">
  Perplexity AI 的 API Key。配置后可在扫描中启用实时 Web 搜索，用于 OSINT 与漏洞研究。
</ParamField>

<ParamField path="STRIX_TELEMETRY" default="1" type="string">
  遥测开关。设置为 `0`、`false`、`no` 或 `off` 可关闭 PostHog、Scarf 与 OTEL 遥测。
</ParamField>

<ParamField path="TRACELOOP_BASE_URL" type="string">
  远程 OpenTelemetry 导出地址。未设置时，Strix 只在本地保留 tracing 数据。
</ParamField>

<ParamField path="TRACELOOP_API_KEY" type="string">
  远程 tracing 导出使用的 API Key。只有同时设置 `TRACELOOP_BASE_URL` 与 `TRACELOOP_API_KEY` 时，远程导出才会启用。
</ParamField>

<ParamField path="TRACELOOP_HEADERS" type="string">
  可选的 OTEL 额外请求头，支持 JSON 对象或 `key=value,key2=value2` 形式。适用于 Langfuse 或自建 OTLP 网关。
</ParamField>

未配置远程 OTEL 变量时，Strix 仍会把完整运行遥测写入本地：

```bash
strix_runs/<run_name>/events.jsonl
```

配置远程变量后，Strix 会同时写入本地 JSONL 和远端 OTEL 端点。

## Docker 配置

<ParamField path="STRIX_IMAGE" default="ghcr.io/usestrix/strix-sandbox:1.0.0" type="string">
  指定用于沙箱容器的 Docker 镜像。
</ParamField>

<ParamField path="DOCKER_HOST" type="string">
  Docker daemon socket 路径，适用于远程 Docker 或自定义运行环境。
</ParamField>

<ParamField path="STRIX_RUNTIME_BACKEND" default="docker" type="string">
  沙箱运行时后端。
</ParamField>

<ParamField path="STRIX_MAX_LOCAL_COPY_MB" default="1024" type="integer">
  本地目录目标允许逐文件复制进沙箱的最大体积，单位为 MB。超过后会提前退出并提示改用 `--mount`。设为 `0` 可关闭该检查。
</ParamField>

## 沙箱配置

<ParamField path="STRIX_SANDBOX_EXECUTION_TIMEOUT" default="120" type="integer">
  沙箱操作的最大执行时间，单位为秒。
</ParamField>

<ParamField path="STRIX_SANDBOX_CONNECT_TIMEOUT" default="10" type="integer">
  连接沙箱容器的超时时间，单位为秒。
</ParamField>

## 配置文件

Strix 默认把配置保存到 `~/.strix/cli-config.json`。你也可以通过 `--config` 指定自定义配置文件：

```bash
strix --target ./app --config /path/to/config.json
```

**配置文件格式：**

```json
{
  "env": {
    "STRIX_LLM": "openai/gpt-5.4",
    "LLM_API_KEY": "sk-...",
    "STRIX_REASONING_EFFORT": "high"
  }
}
```

## 示例配置

```bash
# 必填
export STRIX_LLM="openai/your-compatible-model"
export LLM_API_KEY="your-api-key"

# 可选：兼容网关或本地模型
export LLM_API_BASE="https://your-gateway.example/v1"

# 可选：启用 Web 搜索
export PERPLEXITY_API_KEY="pplx-..."

# 可选：自定义超时
export LLM_TIMEOUT="600"
export STRIX_SANDBOX_EXECUTION_TIMEOUT="300"
```
