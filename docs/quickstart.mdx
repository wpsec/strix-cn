---

`strix-cn` 是面向中文用户的持续维护分支。快速开始会优先展示国内可访问模型、OpenAI-compatible 网关、本地模型，以及中文说明和环境兼容相关的推荐配置。

## 前置要求

- Docker 已启动
- 一个可用的 LLM 接入方式。优先推荐 OpenAI-compatible 网关、LiteLLM 路由或本地模型；仍兼容 OpenAI、Anthropic、Google 等上游提供商。

## 安装

<Tabs>
  <Tab title="curl">
    ```bash
    curl -sSL https://strix.ai/install | bash
    ```
  </Tab>
  <Tab title="pipx">
    ```bash
    pipx install strix-agent
    ```
  </Tab>
</Tabs>

## 配置

优先选择你当前网络环境最容易稳定访问的模型路径：

<Tabs>
  <Tab title="兼容网关">
    ```bash
    export STRIX_LLM="openai/your-compatible-model"
    export LLM_API_KEY="your-api-key"
    export LLM_API_BASE="https://your-gateway.example/v1"
    ```
  </Tab>
  <Tab title="本地模型">
    ```bash
    export STRIX_LLM="ollama/qwen3-vl"
    export LLM_API_BASE="http://localhost:11434"
    ```
  </Tab>
</Tabs>

<Tip>
如果你已经有 OpenAI-compatible 中转、企业网关、LM Studio、vLLM 或 Ollama，这个分支会优先围绕这些接入方式做兼容性维护。
</Tip>

## 执行第一次扫描

```bash
strix --target ./your-app
```

<Note>
首次运行会自动拉取 Docker 沙箱镜像。结果会保存到 `strix_runs/<run-name>`。
</Note>

<Note>
当前分支默认输出中文报告，也支持直接使用中文 `--instruction` 或 `--instruction-file` 描述测试要求。
</Note>

## 支持的目标类型

Strix 支持多种扫描目标：

```bash
# 本地代码库
strix --target ./app-directory

# GitHub 仓库
strix --target https://github.com/org/repo

# 在线 Web 应用
strix --target https://your-app.com

# 多目标（白盒测试）
strix -t https://github.com/org/repo -t https://your-app.com

# 从文件读取目标，每行一个，忽略空行和注释
strix --target-list ./targets.txt
```

## 国内 / 受限网络环境建议

```bash
# 如果安装源或模型接口需要代理，可按需设置
export HTTP_PROXY="http://127.0.0.1:7897"
export HTTPS_PROXY="http://127.0.0.1:7897"
export ALL_PROXY="socks5://127.0.0.1:7897"
```

- 模型接口走兼容网关时，优先通过 `LLM_API_BASE` 指向你当前可访问的地址。
- 如果团队更关注数据不出网，可优先使用 Ollama、LM Studio、vLLM 等本地或内网模型服务。
- 如果本地源码目录包含 symlink 或平台特殊临时目录，这个分支会持续补齐 macOS、Linux、Windows 的兼容性问题。
