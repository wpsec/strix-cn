---
title: "扫描模式"
description: "根据场景选择合适的扫描深度"
---

Strix 提供三种扫描模式，用于平衡速度与完整性。

## Quick

```bash
strix --target ./app --scan-mode quick
```

用于快速发现明显漏洞。适合：
- CI/CD 流水线
- Pull Request 校验
- 快速冒烟测试

**耗时**：数分钟

## Standard

```bash
strix --target ./app --scan-mode standard
```

在速度和覆盖之间取得平衡，适合常规安全评审。适用于：
- 周期性安全检查
- 发布前验证
- 研发阶段里程碑检查

**耗时**：约 30 分钟到 1 小时

**白盒行为**：使用源码感知映射与静态预筛选，优先验证最值得动态利用的路径。

## Deep

```bash
strix --target ./app --scan-mode deep
```

面向更完整的渗透测试流程。适合：
- 全面安全审计
- 生产前评审
- 关键应用安全评估

**耗时**：通常为 1 到 4 小时，取决于目标复杂度

**白盒行为**：执行更广泛的源码感知预筛选（`semgrep`、AST 结构搜索、密钥检查、供应链检查等），随后系统化地对高优先级候选点做动态验证。

<Note>
`deep` 是默认模式。它会更积极地探索边界情况、漏洞链和复杂攻击路径。
</Note>

## 如何选择

| 场景 | 推荐模式 |
|------|----------|
| 每个 PR | Quick |
| 每周扫描 | Standard |
| 大版本发布前 | Deep |
| 漏洞赏金研究 | Deep |
