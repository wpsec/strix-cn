Strix 通过 [LiteLLM](https://docs.litellm.ai/docs/providers) 提供模型兼容层，支持大量 LLM 提供商。`strix-cn` 会优先围绕中文团队常见的三类接入方式持续维护：OpenAI-compatible 网关、可直连的兼容提供商，以及本地 / 内网模型服务。

## 推荐接入路径

| 路径 | 适用场景 | 配置方式 |
|------|----------|----------|
| OpenAI-compatible 网关 | 国内可访问接口、企业统一网关、自建中转 | `openai/<your-model>` + `LLM_API_BASE` |
| 兼容提供商 | 想快速接入现成模型服务 | 参考 OpenRouter、Novita 等页面 |
| 本地模型 | 内网、隐私优先、离线或半离线环境 | `ollama/<model>` 或 `openai/<model>` + 本地 `LLM_API_BASE` |

## OpenAI-compatible 网关

```bash
export STRIX_LLM="openai/your-compatible-model"
export LLM_API_KEY="your-api-key"
export LLM_API_BASE="https://your-gateway.example/v1"
```

这种方式适合：

- 国内可访问的模型网关
- 企业自建统一模型入口
- 已有 OpenAI-compatible 中转或审计层的团队

## 本地模型

你也可以使用 [Ollama](https://ollama.com)、[LM Studio](https://lmstudio.ai) 或其他本地服务：

```bash
export STRIX_LLM="ollama/qwen3-vl"
export LLM_API_BASE="http://localhost:11434"
```

如果你使用 LM Studio、vLLM 或其他 OpenAI-compatible 本地服务，也可以采用：

```bash
export STRIX_LLM="openai/local-model"
export LLM_API_BASE="http://localhost:1234/v1"
```

更多细节可查看 [本地模型指南](/llm-providers/local)。

## 模型格式

统一使用 LiteLLM 的 `provider/model-name` 形式：

```
openai/your-compatible-model
anthropic/claude-sonnet-4-6
vertex_ai/gemini-3-pro-preview
bedrock/anthropic.claude-4-5-sonnet-20251022-v1:0
ollama/qwen3-vl
```

<Tip>
  如果你更关心国内可达性与稳定性，优先从 OpenAI-compatible 网关、本地模型和兼容提供商开始接入，通常比直接依赖单一海外上游更容易落地。
</Tip>
